# Pawline Defense

Direct2D 기반 탑뷰 라인 디펜스 게임 프로토타입입니다. 2026 넥슨 대학생 게임잼 `재밌넥` 2차 과제 주제인 "자신만의 탑뷰 디펜스 게임"에 맞춰, 유닛을 소환해 라인을 밀고 상대 기지를 파괴하는 구조로 구성했습니다.

유명 유닛 소환형 라인 디펜스의 재미 구조를 참고했지만, 캐릭터명, 그래픽, UI, 규칙 수치는 모두 새로 만든 오리지널 구현입니다.

## 실행

빌드 후 아래 파일을 실행합니다.

```powershell
.\package\DawnlineDefense\DawnlineDefense.exe
```

## 빌드

필요 환경:

- Windows 10/11
- Visual Studio 2022 C++ Desktop Development workload
- CMake

빌드와 제출용 zip 생성을 한 번에 실행합니다.

```powershell
.\build.ps1
```

생성 결과:

- `build\Release\DawnlineDefense.exe`
- `package\DawnlineDefense\DawnlineDefense.exe`
- `package\DawnlineDefense.zip`

## 조작

- 타이틀 화면: `START`, `STORY`, `DEMO`, `OPTIONS`, `QUIT` 선택
- 타이틀 단축키: `Enter` / `Space` 시작, `S` 프롤로그 다시 보기, `D` 시연 모드, `O` 옵션, `Esc` / `Q` 종료
- 저장 파일이 하나도 없는 첫 시작에서는 오리지널 우주 프롤로그 크롤이 먼저 재생되고, `SKIP` 버튼 또는 `Space`/`Enter`/클릭으로 넘길 수 있습니다.
- 옵션 화면: 기본 게임 속도, 효과음/BGM 볼륨 슬라이더, 피격 흔들림, 눈부심 줄이기, 화면 안전 여백, 진행 데이터 초기화 설정
- 옵션 단축키: `H` 흔들림 토글, `F` 눈부심 줄이기, `M`/`N` 효과음 볼륨, `B`/`V` BGM 볼륨, `R` 오디오 기본값 복구, 좌우 방향키 또는 `[` / `]` 속도 조절, 위아래 방향키 화면 크기 보정, `A` 자동 맞춤, `X` 진행 초기화 확인, `Esc` / `Backspace` 뒤로
- 스테이지 선택 화면: 진행도에 따라 열린 10개 스테이지 선택, 14종 로스터 중 5종 편성, `기록`/`Shop`/`Start Stage` 클릭
- 브리핑 화면: 선택한 행성, 기믹, 적 조합, 편성 유닛, 활성 시너지, 난이도를 확인한 뒤 `Launch`로 출격
- 브리핑 난이도 단축키: `E` 쉬움, `N` 보통, `H` 어려움
- 스테이지 선택 단축키: `1~9`, `0` 또는 좌우 방향키
- `C`: 유닛/적/스테이지 도감 열기
- `S` 또는 `B`: 상점 열기
- `1` ~ `5`: 편성한 유닛 소환
- 마우스 왼쪽 클릭: 유닛 카드와 버튼 선택
- 전장 위 마우스 좌우 끝 이동: 긴 전장 카메라 스크롤
- 전장 위 마우스 왼쪽 드래그 또는 마우스 휠: 카메라 좌우 이동
- 상단 카메라 HUD 클릭: 해당 전장 위치로 카메라 이동
- `F1`: 제출 영상용 데모 모드 켜기/끄기. 자동 카메라, 자동 소환, 월렛 강화, 캐논 사용을 진행합니다.
- `F2`: 테스트 모드 켜기/끄기
- 테스트 모드에서 `F3`: 자원 충전, `F4`: 보스 소환, `F5`: 즉시 클리어, `F6`: 전체 잠금 해제
- `W`: 지갑 업그레이드
- `Space`: 문빔 캐논 발사
- `[` 또는 `-`: 게임 속도 감소
- `]` 또는 `+`: 게임 속도 증가
- `Esc`: 언제든 메뉴 열기, 계속하기/스테이지 선택/게임 종료
- `P`: 일시정지
- `R`: 재시작

## 구성

- `src/main.cpp`: Windows 진입점
- `src/PawlineGame.h`, `src/PawlineGame.cpp`: 공개 게임 클래스와 얇은 pimpl 래퍼
- `src/PawlineGameImpl.h`: 내부 게임 상태, 공용 구조체, 렌더/전투/입력 함수 선언
- `src/PawlineGameImplCore.cpp`: Win32/Direct2D 초기화, 저장/로드, 리셋, 프레임 업데이트
- `src/PawlineGameImplCombat.cpp`: 적 스폰, 유닛 이동/공격, 투사체, CPU 파티클/VFX 수명 갱신과 수량 제한
- `src/PawlineGameImplInput.cpp`: 키보드/마우스 입력, 상점, 편성, ESC 메뉴 처리
- `src/PawlineGameImplLayout.cpp`: 버튼과 카드의 클릭/렌더 공용 좌표
- `src/PawlineGameImplRender.cpp`: Direct2D 렌더링, 이미지 시트 VFX, 셰이더식 절차적 후처리 패스
- `src/GameData.h`, `src/GameData.cpp`: 유닛/적/스테이지 밸런스 데이터
- `src/framework/DeltaTimer.h`, `src/framework/DeltaTimer.cpp`: QueryPerformanceCounter 기반 델타타임 타이머
- `src/framework/SceneManager.h`, `src/framework/SceneManager.cpp`: 참고 프레임워크처럼 화면을 스택으로 관리할 수 있는 씬 매니저
- `src/framework/AudioManager.h`, `src/framework/AudioManager.cpp`: FMOD 선택 빌드와 WinMM 기본 빌드를 함께 지원하는 효과음, 볼륨, 위치음향 관리
- `shaders/pawline_vfx.hlsl`: 전투 후처리 VFX 설계용 HLSL 셰이더 레퍼런스
- `docs/CODE_STRUCTURE_KO.md`: 코드 구조, 전투 흐름, 렌더링, 사운드, QA 체크리스트를 사람이 읽기 쉽게 정리한 문서

게임 루프는 매 프레임 `DeltaTimer::Tick()`으로 계산한 델타타임을 사용해 이동, 쿨타임, 스폰, 파티클/VFX를 갱신합니다. 옵션의 기본 속도와 전투 중 속도 버튼을 통해 게임 속도는 `0.5x`부터 `3.0x`까지 조절할 수 있습니다.

렌더링은 제출 안정성을 위해 Win32 + Direct2D 파이프라인을 유지합니다. 대신 HLSL 셰이더 레퍼런스에 정리한 스캔라인, 비네트, 크로매틱 에지, 충격 링, 별빛 노이즈 수식을 Direct2D 절차적 VFX 패스로 구현해 전투 화면에 적용했습니다.

파티클의 위치, 속도, 수명은 현재 CPU에서 `m_particles`, `m_rings`, `m_beams`, `m_sparkLines`, `m_imageVfx` 컨테이너로 관리합니다. 실제 그리기는 Direct2D의 GPU 가속 렌더링을 사용하지만, GPU compute shader로 파티클 시뮬레이션을 돌리는 구조는 아닙니다. 렉 유발을 줄이기 위해 각 VFX 컨테이너에는 최대 개수 제한을 두고, 오래된 이펙트부터 제거합니다.

프레임워크는 참고 레포의 `game_framwork`, `scene_manager`, `sound_manager` 구조를 Direct2D 프로젝트에 맞춰 축소 적용했습니다. 현재 게임은 기존 단일 애플리케이션 루프를 유지하지만, `SceneManager`와 `AudioManager`가 프레임워크에 들어가 있어 이후 타이틀/상점/전투 씬 분리가 쉬운 구조입니다.

## 게임 규칙

- 플레이어 기지는 왼쪽, 적 기지는 오른쪽에 있습니다.
- 시간이 지나면 적 유닛이 오른쪽 기지에서 자동으로 등장합니다.
- AI 디렉터가 현재 전선 상태를 보고 적 스폰 간격과 엘리트 등장 확률을 미세 조정합니다.
- 플레이어는 `ENERGY`를 소비해 유닛을 소환합니다.
- 적을 처치하면 추가 `ENERGY`를 얻습니다.
- `월렛`을 업그레이드하면 에너지 최대치와 회복량이 증가합니다.
- `월렛` 레벨이 오르면 유닛 소환 비용이 낮아지고, 소환 쿨타임이 줄어들며, 새로 소환되는 아군의 체력/공격력이 증가합니다.
- `월렛`은 일정 시간마다 보급 펄스를 일으켜 에너지, 캐논 충전량, 아군 회복을 제공합니다.
- 적 기지를 먼저 파괴하면 승리하고, 플레이어 기지가 파괴되면 패배합니다.
- 스테이지를 클리어하면 `LUMEN`을 얻고, 첫 클리어에는 추가 보너스를 받습니다.
- 상점에서 `LUMEN`으로 잠긴 유닛을 구매하거나 보유 유닛을 최대 Lv.5까지 강화할 수 있습니다.
- 유닛이 Lv.5가 되면 `진화` 상태가 되어 이름, 능력치, 전투 중 빛 장식과 공격 잔광이 달라집니다.
- 기본 5종 외의 고급 유닛은 처음에는 잠겨 있으며, 구매 후 편성할 수 있습니다.
- 스테이지는 수성부터 순서대로 열리며, 이전 행성을 클리어해야 다음 행성에 도전할 수 있습니다.
- `전술 기록` 화면에서는 보유 유닛, 적 종류, 스테이지 진행 상태와 기믹을 확인할 수 있습니다.
- 진행 상태는 실행 파일 옆 `pawline_progress_slot1.txt` ~ `pawline_progress_slot3.txt`에 슬롯별로 저장되며, 옵션 화면에서 슬롯 선택/저장/불러오기/삭제를 할 수 있습니다.
- 유닛과 기지는 피격 시 흔들림, 피해 숫자, 파티클로 피드백을 제공합니다.
- 옵션 화면에서 피격 흔들림을 끄고 켤 수 있습니다.
- 옵션 화면에서 효과음 볼륨과 BGM 볼륨을 각각 `0%`부터 `100%`까지 조절할 수 있으며, `AUDIO RESET`으로 기본값을 되돌릴 수 있습니다.
- 플레이어 14종과 적 17종은 서로 다른 공격 효과음을 사용하고, BGM은 우주 분위기의 루프 음악을 낮은 볼륨으로 깔아 전투음을 덮지 않게 했습니다.
- 전투 화면에는 아군/적 공격 wind-up, 타격, 반동 애니메이션과 캐릭터별 광선 빔, 스파크 라인, 화면 플래시, 충격 링이 적용됩니다.
- 유닛 애니메이션은 `Idle`, `Move`, `Windup`, `Attack`, `Recover`, `Hit`, `Death` 상태로 나뉘며, 상태에 따라 보폭, 피격 반동, 공격 전진이 달라집니다.
- 파티클은 점, 글로우, 먼지, 파편, 불씨, 눈 결정, 버블 타입으로 나뉘어 행성 기믹과 처치 연출에 사용됩니다. CPU에서 델타타임으로 갱신하지만, 파티클/링/빔/스파크/이미지 VFX마다 최대 개수 제한을 둬 과도한 누적을 막습니다.
- 카툰풍 외곽선, 액션 라인, 하프톤 점, 스캔라인, 행성별 열기/별빛 후처리로 전투 화면을 렌더링합니다.
- 전투 배경에는 별, 성운, 밝고 어두운 공간 레이어를 추가했고, 공격 효과는 전체 화면 플래시 대신 캐릭터 주변 로컬 조명과 방향성 그림자로 표현합니다.
- 아군 14종과 적 17종은 각자 다른 무기 실루엣과 공격 이펙트를 사용합니다. 예를 들어 방패 돌진, 전기 지팡이, 쌍검, 종소리 파동, 망치 충격, 얼음 창, 드릴, 프리즘 저격, 공허 구체처럼 공격 방식이 분리됩니다.
- 각 행성은 주기적인 스테이지 기믹을 갖습니다. 예를 들어 수성은 열파, 금성은 산성 안개와 사거리 감소, 화성은 운석, 목성은 중력 변동, 토성은 고리 지원군, 태양은 플레어와 보스 압박을 사용합니다.
- 행성 기믹과 보스 패턴은 발동 전 원형/직선/전장 범위 예고가 먼저 표시되어 플레이어가 대응할 수 있습니다.
- 보스 등장 시 카메라가 보스 쪽으로 이동하고 경고 컷인, 보스 체력바, 카메라 반응이 표시됩니다. 보스는 전용 체력/방어 보정과 페이즈 실드를 받아 일반 엘리트보다 오래 버티며, 체력 구간에 따라 공격 속도와 패턴 주기가 강화됩니다.
- 보스는 행성별 고유 패턴을 사용하며, 수성은 압축 파동, 금성은 산성 장막, 화성은 붉은 낙하, 토성은 고리 창 소환, 태양은 삼중 플레어처럼 서로 다른 공격을 합니다.
- 편성 조합에 따라 `Guard Wall`, `Arc Focus`, `Rush Pack`, `Star Scope`, `Mint Supply`, `Sun Chime` 같은 시너지가 활성화됩니다.
- 상점에서는 선택한 유닛의 체력, 공격력, 역할, 레벨 상태를 상세 패널로 확인할 수 있습니다.
- 브리핑 화면은 편성 전력과 스테이지 위협도를 비교해 상점 강화 또는 난이도 도전을 추천합니다.
- 결과 화면은 랭크/메달과 함께 구매 또는 강화 추천을 보여주며, 태양 최종 클리어 시 캠페인 클리어 연출을 표시합니다.
- 태양까지 모두 클리어하면 짧은 엔딩 장면이 먼저 재생되고, 이후 결과 화면에서 보상과 랭크를 확인할 수 있습니다.
- 첫 전투 초반에는 유닛 소환, 지갑, 카메라, 문빔 캐논을 단계별로 알려주는 튜토리얼 패널이 표시됩니다.
- 전장은 한 화면보다 넓은 월드로 구성되며, 마우스 가장자리 이동/드래그/휠로 좌우를 둘러볼 수 있습니다.
- 상단 카메라 HUD는 현재 보고 있는 구간, 양쪽 기지, 유닛 위치를 간략히 보여줍니다.
- 인게임 HUD는 직접 그린 픽셀 블록 글자, 카툰 패널, 커맨드 패널로 구성되며 버튼과 카드 클릭 시 UI 파동이 표시됩니다.
- 창 크기와 모니터 작업 영역에 맞춰 게임 화면이 자동 축소/중앙 정렬되며, 옵션의 `화면 안전 여백`으로 잘림 현상을 추가 보정할 수 있습니다.
- 스테이지 밸런스는 `GameData.cpp` 안의 `kStageTable`에서 테이블 형태로 관리해 체력, 시작 에너지, 적 등장 간격, 위협도, 보스 등장 시점을 한 곳에서 조정할 수 있습니다.

## 유닛

- `푸른검병`: 저렴하고 빠르게 뽑을 수 있는 기본 근접 유닛입니다.
- `방패병`: 느리지만 체력이 높아 전선을 버티는 탱커입니다.
- `전격술사`: 비싸지만 사거리가 긴 원거리 유닛입니다.
- `질주병`: 빠르게 전선을 밀어붙이는 근접 유닛입니다.
- `종술사`: 중거리에서 안정적으로 피해를 주는 원거리 유닛입니다.
- `거인병`: 비싸지만 체력과 한 방 피해가 높은 대형 유닛입니다.
- `빙결방패`: 방패형 근접 유닛으로 전선을 안정적으로 버팁니다.
- `혜성병`: 매우 빠르게 달려가 초반 전선을 밀어주는 근접 유닛입니다.
- `궤도드론`: 긴 사거리와 높은 단발 피해를 가진 후방 원거리 유닛입니다.
- `태양검`: 비싸지만 체력과 공격력이 모두 높은 최종형 근접 유닛입니다.
- `지원병`: 중거리에서 안정적으로 지원 사격하는 원거리 유닛입니다.
- `드릴기갑`: 전선을 뚫고 들어가는 고화력 근접 유닛입니다.
- `프리즘술사`: 긴 사거리와 높은 단발 피해를 가진 저격형 유닛입니다.
- `성운포대`: 비싸지만 강력한 후방 원거리 최상위 유닛입니다.

## 스테이지

- `수성`: 먼지형/침형 적이 등장하며, 회색 크레이터 지형으로 표현했습니다.
- `금성`: 산성사수와 거울사수가 등장하며, 짙은 대기와 산성 구름 밴드가 깔립니다.
- `지구`: 포자병과 포자포병 균형 웨이브가 등장하며, 바다색 지형과 녹색 대륙, 구름이 보입니다.
- `화성`: 녹슨 장갑형 적이 합류하며, 붉은 모래 능선과 균열이 배경을 이룹니다.
- `목성`: 중력방패와 지진돌격 중장갑 적의 압박이 강하고, 가스 밴드와 거대한 폭풍이 나타납니다.
- `토성`: 고리사수 원거리 적이 등장하며, 고리 궤도가 전장 위를 가로지릅니다.
- `천왕성`: 얼음러너와 원거리 압박이 강하고, 기울어진 얼음 궤도와 청록색 빙면을 사용합니다.
- `해왕성`: 해류사수 원거리 러시가 거세며, 먼 푸른 파도와 소용돌이를 강조했습니다.
- `명왕성`: 공허/지진 중장갑 적과 빠른 보스 타이밍이 특징이며, 어두운 얼음 지형과 작은 별빛을 배치했습니다.
- `태양`: 플레어 돌격형, 혜성형 적과 최종 보스를 상대하며, 태양광과 플레어가 전장을 덮습니다.

## 에셋 및 라이선스

효과음은 Kenney의 무료 CC0 음원 팩을 사용했습니다. `build.ps1`은 로컬 FMOD SDK를 자동 감지해 `-DUSE_FMOD=ON -DFMOD_SDK_DIR=...`로 빌드하고, 실행에 필요한 `fmod.dll`을 패키지에 복사합니다. SDK가 없을 때만 WinMM 기반 위치 볼륨 보정으로 실행됩니다. `assets/sfx/KENNEY_AUDIO_LICENSE.md`와 `docs/FMOD_RUNTIME_NOTICE.md`에 출처와 변환 이유를 정리했습니다.

- Kenney Sci-fi Sounds: `https://kenney.nl/assets/sci-fi-sounds`
- Kenney Digital Audio: `https://kenney.nl/assets/digital-audio`
- Kenney Impact Sounds: `https://kenney.nl/assets/impact-sounds`
- Kenney Interface Sounds: `https://kenney.nl/assets/interface-sounds`
- Kenney RPG Audio: `https://kenney.nl/assets/rpg-audio`

배경음악은 OpenGameArt의 CC0/Public Domain 루프 음원을 사용했습니다. 실제 파일과 출처는 `assets/music/MUSIC_LICENSE.md`에 정리했습니다.

- Outer Space Loop by wipics: `https://opengameart.org/content/outer-space-loop`

그래픽은 Direct2D 절차적 렌더링과 외부 무료 에셋을 게임 규격에 맞게 가공한 스프라이트를 함께 사용합니다. 전투 무기는 `assets/weapons/external_roles`의 역할별 근접/원거리 PNG를 사용하며, RgsDev/OpenGameArt, Tom L/OpenGameArt, zeroisnotnull/OpenGameArt, Kenney의 CC0 에셋 출처를 해당 폴더의 `LICENSE_NOTE.md`에 정리했습니다. 스테이지 배경은 NASA Image and Video Library 이미지를 `1600x900` 게임 배경으로 보정해 사용하며, 출처는 `assets/backgrounds/NASA_SOURCES.md`에 정리했습니다. 화면 글꼴에는 프로젝트에 포함된 Galmuri 폰트를 사용하며, 해당 라이선스는 `assets/fonts/Galmuri-OFL.md`에 있습니다.
