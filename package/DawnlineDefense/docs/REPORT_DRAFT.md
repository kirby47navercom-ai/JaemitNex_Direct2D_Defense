# 작업물 리포트 초안

## 1. 결과물 설명

`Pawline Defense`는 Win32와 Direct2D로 제작한 탑뷰 라인 디펜스 게임입니다. 플레이어는 타이틀 화면에서 시작한 뒤, 수성부터 태양까지 이어지는 10개 스테이지 중 하나를 고르고, 14종 로스터 중 5종을 편성한 뒤 전투에 들어갑니다. 전투에서는 에너지를 모아 유닛을 소환하고, 오른쪽에서 몰려오는 적 유닛을 막아내며 상대 기지를 파괴해야 합니다.

유닛 카드, 자원 수급, 기지 대 기지 전투처럼 유명 라인 디펜스 장르에서 검증된 구조를 참고했지만, 캐릭터와 명칭, 그래픽, 밸런스는 모두 새로 제작했습니다.

핵심 플레이는 다음과 같습니다.

- 유닛 카드 쿨타임과 에너지 비용 관리
- 14종 캐릭터 중 5종을 선택하는 편성 시스템
- 스테이지 클리어 보상 `LUMEN`을 이용한 유닛 구매와 Lv.5까지의 강화 시스템
- 전투 중 월렛 업그레이드에 따른 소환 비용 할인, 쿨타임 감소, 아군 능력치 강화, 주기적 보급 펄스
- 수성, 금성, 지구, 화성, 목성, 토성, 천왕성, 해왕성, 명왕성, 태양으로 구성된 10개 스테이지
- 지갑 업그레이드를 통한 에너지 최대치와 회복량 성장
- 근접 기본 유닛, 탱커 유닛, 원거리 유닛의 조합
- 행성별로 달라지는 먼지형, 황산형, 이끼형, 포자형, 장갑형, 폭풍형, 지진형, 고리 창병, 빙결형, 파도형, 공허형, 플레어형, 혜성형 적의 압박
- 충전형 문빔 캐논을 활용한 위기 대응
- `시작하기`, `옵션`, `나가기`를 갖춘 시작 화면과 언제든 열 수 있는 `Esc` 메뉴를 통한 완성된 게임 흐름
- 전투 중 `0.5x`부터 `3.0x`까지 조절 가능한 게임 속도
- 한 화면보다 긴 전장을 마우스 가장자리 이동, 드래그, 휠로 둘러보는 카메라 시스템
- 상단 카메라 HUD를 통한 현재 시야, 양쪽 기지, 유닛 위치 확인
- 옵션 화면에서 조절 가능한 기본 게임 속도, 피격 흔들림 토글, 화면 안전 여백
- 수성의 크레이터, 금성의 대기 밴드, 지구의 바다와 대륙, 목성의 폭풍, 토성의 고리, 태양의 플레어처럼 스테이지별로 달라지는 전장 배경
- 아군/적 공통 공격 wind-up, 타격, 반동 애니메이션과 캐릭터별 공격 실루엣, 투사체 광선, 충격파 링, 스파크 라인, 캐논 화면 플래시, 강화/구매 연출 등 강화된 시각 피드백
- 별, 성운, 밝고 어두운 공간 레이어를 사용한 우주 전장과 공격 중 캐릭터 주변에만 생기는 로컬 조명/방향성 그림자
- 아군 14종과 적 17종이 각자 다른 무기 실루엣, 공격 모션, 공격 이펙트를 갖도록 분리한 전투 표현
- 직접 그린 5x7 픽셀 블록 글자, 카툰 패널, 커맨드 패널, 버튼/카드 클릭 UI 파동과 hover 빛 효과로 강화된 입력 피드백
- 전투 전 브리핑 화면, 초반 튜토리얼 패널, 결과 화면 랭크/메달 표시를 통한 완성된 플레이 흐름
- 행성별 열파, 산성 안개, 보급 개화, 운석, 중력 변동, 고리 지원군, 빙결 돌풍, 파도, 공허, 태양 플레어 같은 실제 스테이지 기믹
- 보스 등장 컷인, 보스 체력바, 카메라 포커스 이동, 큰 충격 시 카메라 반응
- `Idle`, `Move`, `Windup`, `Attack`, `Recover`, `Hit`, `Death` 상태로 나뉜 유닛 애니메이션 상태머신과 타입별 파티클 시스템
- HLSL 셰이더 레퍼런스에 설계한 스캔라인, 비네트, 크로매틱 에지, 충격 링 수식을 Direct2D 절차적 VFX 패스로 구현하고, 카툰 외곽선/하프톤/액션 라인 후처리를 추가한 연출
- 다른 PC/모니터에서 화면이 잘리는 상황을 줄이기 위한 자동 축소, 중앙 정렬, 옵션 화면 안전 여백 조절

## 2. 제작 과정

1. 과제 요구사항을 분석해 `실행 가능한 탑뷰 디펜스`를 우선 목표로 정했습니다.
2. 사용자가 요청한 유닛 소환형 라인 디펜스 방향을 반영하되, 원작 캐릭터를 복제하지 않는 오리지널 콘셉트로 재설계했습니다.
3. Win32 메시지 루프와 Direct2D 렌더 타깃을 구성했습니다.
4. 외부 에셋 저작권 리스크를 줄이기 위해 그래픽을 Direct2D 도형 렌더링으로 제작했습니다.
5. 기지, 라인, 유닛 카드, 지갑, 캐논, 적 스폰 디렉터를 구현했습니다.
6. 타이틀 화면, 옵션 화면, 10개 스테이지 선택, 캐릭터 편성, 결과 화면을 추가해 플레이 흐름을 완성했습니다.
7. 게임 루프에서 유닛 이동, 자동 타깃팅, 근접/원거리 공격, 투사체 충돌, 기지 피해, 보상 지급을 연결했습니다.
8. 전투 중 게임 속도를 낮추거나 높일 수 있도록 델타타임에 속도 배율을 적용했습니다.
9. 캐릭터 풀을 14종으로 늘리고, 전투 중 사용할 수 있는 편성 슬롯도 5칸으로 확장했습니다.
10. 스테이지 클리어 보상 자원 `LUMEN`, 첫 클리어 보너스, 상점 구매, 유닛 강화 시스템을 추가했습니다.
11. 구매, 강화, 클리어 상태가 실행 파일 옆 저장 파일에 유지되도록 진행 저장을 추가했습니다.
12. 각 행성 스테이지가 서로 다르게 보이도록 Direct2D 도형 기반 배경 장식을 추가했습니다.
13. 스테이지별 적 조합과 강화형 적이 달라지도록 적 스폰 디렉터를 수정했습니다.
14. 전투 중이나 선택 화면에서 `Esc` 메뉴를 열어 계속하기, 스테이지 선택, 게임 종료를 할 수 있도록 했습니다.
15. 투사체 잔광, 타격 충격파, 구매/강화 링 이펙트를 추가해 전투와 성장 피드백을 강화했습니다.
16. `main.cpp`, 공개 래퍼 `PawlineGame`, 내부 구현 `PawlineGameImpl`, `GameData`, `DeltaTimer`로 파일을 나누고, QueryPerformanceCounter 기반 델타타임으로 업데이트되도록 정리했습니다.
17. 커진 내부 구현을 `Core`, `Combat`, `Input`, `Layout`, `Render` 파일로 추가 분리하고, 주요 루프와 전투/VFX 흐름에 설명 주석을 붙였습니다.
18. 셰이더식 VFX 설계 파일 `shaders/pawline_vfx.hlsl`을 작성하고, Direct2D 렌더러 안에서 같은 의도의 절차적 후처리 레이어를 구현했습니다.
19. 캐릭터별 공격 애니메이션 상태를 추가해 유닛마다 공격 시 전진, 반동, 잔상, 충격파, 궤도 확장, 드릴 돌진처럼 다른 시각 동작을 갖게 했습니다.
20. 월렛 업그레이드가 단순 에너지 최대치 증가에 그치지 않도록 소환 비용 할인, 쿨타임 감소, 유닛 능력치 상승, 보급 펄스 시스템을 추가했습니다.
21. 전장 폭을 한 화면보다 길게 확장하고, 마우스 가장자리 이동/드래그/휠로 좌우를 둘러볼 수 있는 카메라를 추가했습니다.
22. 카메라 HUD와 UI 클릭 파동, 버튼 hover 글로우를 추가해 조작 피드백을 강화했습니다.
23. 다른 PC에서 실행했을 때 화면이 잘릴 수 있는 문제를 고려해 창 크기를 작업 영역에 맞춰 자동 조정하고, 렌더링 좌표를 전체 스케일/중앙 정렬 방식으로 보정했습니다.
24. 옵션 화면에 `화면 안전 여백` 조절을 추가해 사용자가 직접 화면 크기를 줄이거나 자동 맞춤으로 되돌릴 수 있게 했습니다.
25. 공격 애니메이션을 단순 흔들림에서 wind-up, 타격, 반동 곡선으로 확장하고, 아군/적 모두에게 카툰 액션 라인과 팔/무기 스윙을 추가했습니다.
26. 인게임 HUD 글씨가 배경과 따로 노는 문제를 줄이기 위해 외곽선 텍스트, 카툰 패널, 커맨드 패널, 굵은 버튼 외곽선을 추가했습니다.
27. Direct2D 후처리 패스에 하프톤 점, 전투 속도선, 굵은 잉크 프레임, 공격 중 주변 링을 추가해 카툰 그래픽 톤을 강화했습니다.
28. 상단 전투 로고를 일반 폰트 텍스트에서 Direct2D 도형으로 직접 그린 픽셀 블록 로고와 배지로 교체했습니다.
29. 전투 배경을 더 우주처럼 보이도록 별, 성운, 공간 리본, 위쪽 광원/아래쪽 그림자 레이어로 재구성했습니다.
30. 일반 공격 시 전체 화면이 밝아지는 연출을 제거하고, 캐릭터 주변 로컬 조명과 오른쪽 아래로 떨어지는 방향성 그림자를 추가했습니다.
31. 아군 14종의 무기와 공격 이펙트를 방패, 전기 지팡이, 쌍검, 종, 망치, 얼음 창, 혜성 랜스, 궤도 구체, 태양검, 회복 지팡이, 드릴, 프리즘 저격, 성운 구체 등으로 분리했습니다.
32. 적 17종도 클로, 둔기, 침창, 황산 포자, 덩굴, 녹슨 칼날, 폭풍 방패, 고리 창, 얼음 낫, 파도 삼지창, 공허 구체, 화염 칼날, 포자탄, 지진 망치, 반사 방패, 혜성 창, 보스 광선 등으로 차이를 만들었습니다.
33. 인게임 HUD의 주요 영어/숫자 텍스트를 DirectWrite 기본 글자가 아니라 Direct2D 사각형으로 직접 찍는 5x7 픽셀 블록 글자로 교체했습니다.
34. 유닛 애니메이션을 명시적인 상태머신으로 정리하고, 파티클을 점/글로우/먼지/파편/불씨/눈/버블 타입으로 나눠 행성 기믹과 처치 연출을 풍부하게 만들었습니다.
35. 스테이지마다 실제 전투 규칙이 달라지는 주기 기믹을 추가했습니다. 수성은 열파, 금성은 산성 안개와 사거리 감소, 지구는 보급 개화, 화성은 운석, 목성은 중력 변동, 토성은 고리 지원군, 태양은 플레어와 보스 압박을 사용합니다.
36. 스테이지 선택 후 바로 전투로 들어가지 않고, 행성 정보/기믹/적 조합/편성 유닛을 확인하는 브리핑 화면을 추가했습니다.
37. 첫 전투 초반에는 소환, 월렛, 카메라, 문빔 캐논을 알려주는 짧은 튜토리얼 패널을 표시하고, 결과 화면에는 랭크와 메달을 추가했습니다.
38. 보스 등장 시 경고 컷인과 보스 체력바, 카메라 포커스 이동, 카메라 흔들림을 추가해 후반 스테이지의 긴장감을 강화했습니다.
39. 스테이지 수치를 `GameData.cpp`의 `kStageTable`로 테이블화하고, `docs/BALANCE_TABLES.md`에 검토용 밸런스 표를 추가했습니다.
40. CMake와 PowerShell 빌드 스크립트를 추가해 실행 파일과 제출용 zip을 생성할 수 있도록 했습니다.

## 3. 사용한 AI와 용도

- OpenAI Codex/GPT 계열 AI를 사용했습니다.
- 사용 목적:
  - Direct2D 기반 Win32 프로젝트 구조 설계
  - 유닛 소환형 탑뷰 라인 디펜스 규칙과 밸런스 초안 구성
  - 행성 테마 스테이지 구성, 배경 연출, 적 조합, 성장 구조, 난이도 곡선 설계
  - C++ 게임 루프, 렌더링, 입력 처리, 유닛 전투 코드 작성 보조
  - 파일 분리, pimpl 기반 게임 클래스, 델타타임 타이머 구조 정리
  - HLSL 셰이더 레퍼런스와 Direct2D 절차적 VFX 연출 설계
  - 제출용 README와 리포트 초안 작성
  - 빌드 오류 탐지와 수정 방향 제안

AI가 생성한 내용을 그대로 방치하지 않고, 과제 목적에 맞는 실행 가능성, 외부 에셋 라이선스, 조작 명확성, 제출 구조를 기준으로 검토하며 반영했습니다.

## 4. 어려웠던 점과 해결 과정

### 기존 IP와 닮은 재미를 오리지널로 바꾸는 문제

요청 방향은 유닛 소환형 라인 디펜스였지만, 특정 게임의 캐릭터나 이름을 그대로 쓰면 저작권 문제가 생길 수 있습니다. 그래서 "유닛 카드로 소환한다", "에너지를 모아 전선을 민다", "상대 기지를 부순다"는 구조만 참고하고, 캐릭터명과 도형 그래픽은 모두 새로 만들었습니다.

### Direct2D만으로 게임 화면의 가독성을 확보하는 문제

외부 이미지 에셋 없이 모든 그래픽을 도형으로 표현해야 했기 때문에, 적/타워/길/슬롯이 한눈에 구분되도록 색상과 두께를 조정했습니다. 경로는 굵은 선을 겹쳐 그려 깊이를 만들고, 타워는 타입별 색상과 포신 방향으로 구분했습니다.

### 스테이지별 배경 차이를 만드는 문제

처음에는 스테이지마다 색만 달라 보여 행성 테마가 약했습니다. 이를 보완하기 위해 스테이지 인덱스에 따라 전장 배경과 라인 패턴을 다르게 그리도록 분리했습니다. 수성은 크레이터와 균열, 금성은 대기 밴드, 지구는 대륙과 구름, 목성은 가스 밴드와 폭풍, 토성은 고리, 태양은 광선과 플레어를 Direct2D 도형으로 표현했습니다.

### 스테이지별 적이 비슷해 보이는 문제

초기 버전에서는 난이도는 달라졌지만 적 종류가 거의 같아 스테이지 변화가 약했습니다. 이를 해결하기 위해 적 타입을 4종에서 17종으로 늘리고, 스테이지별로 주력 적과 강화형 적이 다르게 나오도록 스폰 로직을 분리했습니다. 예를 들어 금성은 황산 안개형과 반사형 원거리 적, 목성은 폭풍 방패형과 지진형 중장갑 적, 천왕성은 빙결 질주형, 태양은 플레어 돌격형과 혜성형 적, 최종 보스가 중심이 됩니다.

### 클리어 이후 반복 플레이 동기 부족 문제

단순히 스테이지를 고르고 전투를 반복하는 구조만으로는 장기 목표가 약했습니다. 그래서 스테이지를 클리어하면 `LUMEN`을 얻고, 첫 클리어 보너스를 통해 초반 상점 구매가 가능하도록 설계했습니다. 기본 5종 외의 유닛은 잠금 상태로 시작하고, 상점에서 구매하거나 보유 유닛을 Lv.5까지 강화할 수 있게 만들어 반복 플레이의 목적을 추가했습니다.

### 월렛 업그레이드의 의미가 약한 문제

초기 월렛은 에너지 최대치와 회복량만 올려 플레이어가 체감하기 어려웠습니다. 이를 해결하기 위해 월렛 레벨이 오를수록 유닛 소환 비용이 할인되고, 카드 쿨타임이 줄고, 새로 소환되는 아군 유닛의 체력과 공격력이 상승하도록 바꿨습니다. 또한 일정 시간마다 보급 펄스가 발생해 에너지와 캐논 충전량을 회복하고, 아군 유닛을 치료하는 시각적/전략적 효과를 추가했습니다.

### 디펜스 게임의 핵심 피드백 구성

적이 맞고 죽는 과정이 밋밋하면 플레이 감각이 약해지기 때문에, 피해 숫자, 처치 보상 텍스트, 파티클, 체력바를 추가했습니다. 유닛끼리 충돌하거나 공격이 명중하면 공격자와 피격자가 짧게 흔들리도록 만들어 타격감을 보강했습니다. 이후 투사체 잔광과 타격 충격파 링, 캐논 대형 링, 상점 구매/강화 링 이펙트를 추가해 시각 피드백을 더 명확하게 만들었습니다. 동시에 옵션 화면에서 피격 흔들림을 끌 수 있게 해, 강한 화면 움직임이 부담스러운 경우도 고려했습니다.

이후 유닛별 공격 애니메이션 상태를 추가해 캐릭터마다 다른 공격 실루엣을 갖도록 개선했습니다. 예를 들어 Dash/Comet 계열은 잔상을 남기며 전진하고, Titan/Box 계열은 찍기 충격파를 만들고, Spark/Prism/Nebula 계열은 원거리 반동과 빔/궤도 확장 연출을 사용합니다.

이후 공격 모션을 한 번 더 개선해 공격 시작 전에는 몸이 뒤로 당겨지고, 타격 순간에는 앞으로 튀어나가며, 끝에는 짧게 반동이 오도록 곡선을 나눴습니다. 아군과 적 모두에게 공통 액션 라인, 팔/클로 스윙, 잉크 외곽선, 공격 중 실루엣 변형을 적용해 어느 쪽이 공격 중인지 더 명확하게 보이도록 했습니다.

추가 개선에서는 공통 공격선과 공통 무기 느낌을 줄이고, 유닛 타입마다 무기와 이펙트를 별도 함수로 나눴습니다. 예를 들어 `Spark Nya`는 지팡이 끝 전기 구체와 번개 빔을 사용하고, `Box Guard`는 방패를 앞으로 밀며 충격 링을 만들고, `Drill Mew`는 회전 드릴과 나선형 선을 사용합니다. 적도 타입별로 둔기, 포자탄, 반사 방패, 공허 구체, 지진 망치처럼 서로 다른 실루엣을 갖게 했습니다.

### 셰이더 기반 연출과 제출 안정성 사이의 선택

전체 렌더러를 D3D11 스왑체인 기반으로 갈아엎으면 HLSL 후처리를 직접 실행할 수 있지만, 과제 기간 안에 안정적인 실행 파일을 제출하는 것이 더 중요했습니다. 그래서 HLSL 파일에는 후처리 수식을 레퍼런스로 작성하고, 실제 게임에는 동일한 의도의 스캔라인, 비네트, 크로매틱 에지, 별빛 노이즈, 충격 플래시를 Direct2D 절차적 패스로 구현했습니다. 이 방식으로 빌드 안정성을 유지하면서도 셰이더식 화면 효과를 적용했습니다.

카툰 그래픽 느낌을 강화하기 위해 Direct2D 후처리 패스에 굵은 잉크 프레임, 가장자리 하프톤 점, 전투 중 속도선, 공격 유닛 주변 강조 링을 추가했습니다. UI도 같은 톤으로 보이도록 버튼과 카드에 두꺼운 외곽선과 하이라이트를 넣고, 인게임 안내 문구는 외곽선 텍스트와 커맨드 패널 안에 넣었습니다.

이후 화면 전체가 공격 때마다 번쩍이는 느낌을 줄이고, 실제로 빛이 닿는 부분과 그림자가 지는 부분이 보이도록 수정했습니다. 일반 타격에서는 `m_screenFlash`를 사용하지 않고, 공격자와 타격 지점 주변에만 색상 글로우를 그립니다. 동시에 모든 유닛 아래에는 왼쪽 위 광원을 기준으로 오른쪽 아래로 밀리는 타원형 그림자를 그려 공간감을 만들었습니다.

인게임 UI의 주요 정보도 일반 폰트 대신 5x7 픽셀 블록 글자로 직접 그리도록 바꿨습니다. 영어/숫자 중심의 전투 HUD, 유닛 카드, 비용, 키 안내, 속도, 상단 자원 표시는 Direct2D 사각형 블록으로 구성되어 게임 화면에 더 자연스럽게 섞이도록 했습니다.

### 긴 전장과 카메라 조작 문제

초기 전투 화면은 양쪽 기지가 한 화면 안에 거의 모두 보여 거리감이 약했습니다. 이를 보완하기 위해 전장 월드 폭을 확장하고, 렌더링 단계에서 월드 좌표를 카메라만큼 이동시키는 방식으로 좌우 스크롤을 구현했습니다. 마우스를 전장 좌우 끝으로 가져가거나, 전장 위에서 드래그하거나, 마우스 휠을 굴리면 카메라가 이동합니다. 상단에는 현재 시야와 양쪽 기지, 유닛 위치를 보여주는 카메라 HUD를 추가해 긴 전장에서도 상황을 놓치지 않도록 했습니다.

### 실행 환경마다 화면이 잘리는 문제

다른 PC나 배율 설정이 다른 모니터에서 실행하면 1280x800 기준 UI가 화면 밖으로 밀릴 수 있습니다. 이를 줄이기 위해 시작 창 크기를 Windows 작업 영역 안에 들어오도록 제한하고, Direct2D 렌더 타깃 크기에 맞춰 전체 가상 화면을 자동 축소/중앙 정렬하도록 수정했습니다. 그래도 작업 표시줄, 디스플레이 배율, 캡처 환경에 따라 여백이 더 필요한 경우가 있으므로 옵션 화면에 `화면 안전 여백` 조절을 추가했습니다. 사용자는 값을 낮춰 전체 게임 화면을 더 작게 만들 수 있고, `자동 맞춤`으로 기본값에 되돌릴 수 있습니다.

### 한 파일에 커지는 코드를 정리하는 문제

초기 구현은 빠른 완성을 위해 한 파일에 기능을 모았지만, 이후 유지보수성과 설명 가능성을 위해 `main.cpp`, `PawlineGame`, `GameData`, `DeltaTimer`로 분리했습니다. `main.cpp`는 진입점만 담당하고, `PawlineGame`은 게임 상태와 렌더링을, `GameData`는 유닛/적/스테이지 밸런스를, `DeltaTimer`는 프레임 독립적인 델타타임 계산을 담당하도록 역할을 나눴습니다.

이후 `PawlineGame` 내부 구현도 다시 나누어 `Core`는 초기화와 저장/로드, 프레임 업데이트를, `Combat`은 전투 시뮬레이션과 VFX 갱신을, `Input`은 화면별 입력 처리를, `Layout`은 UI 좌표를, `Render`는 Direct2D 그리기와 셰이더식 후처리 패스를 담당하게 했습니다. 핵심 함수에는 델타타임, 타깃 탐색, 투사체 추적, 후처리 패스가 어떻게 연결되는지 설명하는 주석을 추가했습니다.

### 제출 가능한 형태 만들기

단순히 소스 코드만 만드는 것이 아니라, `build.ps1`을 통해 실행 파일과 제출용 zip을 자동으로 생성하도록 구성했습니다. 외부 에셋을 쓰지 않아 라이선스 확인 부담도 줄였습니다.

## 5. 향후 개선 아이디어

- 적 등장 패턴을 더 정교한 스테이지 곡선으로 설계
- 보스별 고유 패턴 추가
- 유닛별 진화/강화 분기
- 사운드와 추가 이펙트로 타격감 강화
- 실제 플레이 테스트 후 난이도 밸런스 조정
