## 사람이 판단한 부분

- 과제 주제에 맞춰 탑뷰 디펜스 장르로 범위를 제한
- 장르 구조만 참고하고 캐릭터명, 스테이지명, 밸런스, 규칙은 오리지널로 구성하는 방향 선택
- 외부 에셋은 CC0/OFL/NASA 출처가 확인된 것만 포함하고, 라이선스 문서를 패키지에 함께 두는 방향 선택
- 제출물 구조를 실행 파일, 문서, zip으로 구성
- 유닛 종류, 적 종류, 행성 스테이지 구성, 지갑 성장, 캐논, 조작 방식 선택
- 시작 화면, 옵션 화면, 상점 화면, ESC 메뉴, 결과 화면, 캐릭터 편성 화면의 구성 선택
- 빌드된 게임을 직접 실행하며 UI 글자 겹침, 화면 잘림, 보스 체력바와 전투 로그 충돌, 효과음 크기 문제를 확인하고 수정 방향 결정
- 기능을 계속 늘리는 대신 제출 안정성, 200MB 용량 제한, 실행 가능성을 기준으로 최종 범위 결정

## 제출 리포트에 넣을 수 있는 요약

이번 작업에서는 AI를 단순 코드 생성기가 아니라 게임 제작 보조자로 활용했습니다. 먼저 과제 요구사항을 기반으로 실행 가능한 범위를 정하고, AI의 도움을 받아 Direct2D 기반 프로젝트 구조와 핵심 게임 루프를 빠르게 구성했습니다. 이후 유닛 카드 소환, 지갑 업그레이드, 적 자동 스폰, 유닛 전투, 투사체 충돌, 기지 파괴, 시작/옵션/상점/ESC/브리핑/결과/도감 UI, 행성 스테이지 선택과 순차 잠금, 행성별 배경과 실제 전투 기믹, 위험 범위 예고, 행성별 보스 패턴/2페이즈, 14종 캐릭터 편성, 유닛 진화, 편성 시너지, 난이도 선택, 데모 자동 전투, 브리핑 밸런스 판정, AI 디렉터, 스테이지 클리어 보상, 유닛 구매와 강화, 진행 저장, 게임 속도 조절, 화면 안전 여백 조절, 캐릭터별 무기/공격 이펙트, 상태머신 기반 애니메이션, 보스 컷인과 카메라 반응, 태양 최종 클리어 연출, 델타타임 기반 업데이트 구조를 하나의 플레이 가능한 결과물로 통합했습니다.

다만 최종 기능 선택, 사람 눈으로 보는 UI/전투 감각 검수, 외부 에셋 라이선스 확인, 제출 패키지 용량 조정, 실행 안정성 판단은 직접 기준을 세워 결정했습니다. AI는 구현 속도를 높이고 여러 대안을 비교하는 도구로 사용했고, 최종 결과물은 반복 실행과 수정 과정을 통해 직접 다듬었습니다.
